§6.5 — §6.6 · Modelo polimórfico y autorización

FunctionalObject: tratar lo heterogéneo de manera homogénea.

Esquema conceptual
ObjectType (id, uuid, name)
  ^ 1
  | N
FunctionalObject (id, uuid, owner,
  created/updated, attrs)
  ^                   ^
  | hereda         | referido por
[Collection, Screen, ...]  Annotation* / ACL*

Authorizable (name, uuid)
  ^      ^     ← herencia polimórfica
Identity   Role
Por qué importa
Tratamiento polimórfico
Anotaciones, ACLs y colecciones se refieren a «cualquier objeto funcional» sin conocer su clase concreta.
Permisos por tipo o por instancia
ACL + ACLExpression + ACLDetail permiten expresar permisos a nivel de ObjectType o de FunctionalObject concreto.
Habilita el oneOf del bundle
Cuando una clase es raíz de jerarquía, el esquema emite un oneOf con sus subclases y un x-discriminator deducido del object_type_id.

§6.3 · Sistema de plugins y extensibilidad · RNF-01

El dominio entra desde fuera: el núcleo no lo conoce.

El contrato · UnibackPlugin
Ciclo de vida
on_initconfig y servicios de terceros get_model_modulesmodelos ORM antes de mapear get_routersrouters FastAPI a montar on_seedfilas por defecto (auto-seed)
Extensiones inyectables
get_field_widgetspistas x-* para el esquema get_importers / exportersformatos del insertador genérico get_fk_resolversresolución de claves foráneas
Descubrimiento automático: el PluginManager escanea src/plugins al arrancar — sin registro manual. «El último registrado para un mismo name gana.»
Ejemplo · el plugin de plantas
class PlantsPlugin(UnibackPlugin):
    name = "PlantsApp"

    # modelo Plant (subclase de FunctionalObject)
    def get_model_modules(self):
        return ["plants.models"]

    # CRUD REST + /schema.json + /bulk
    def get_routers(self):
        return [make_simple_rest_crud(Plant, "plants")]

    # pista de UI para el mapa
    def get_field_widgets(self):
        return [MapCoordinatesWidget()]
Sin tocar el núcleo, la entidad aparece en /api/sys/entities, entra en el bundle, publica su schema.json y <ngt-dynamic-form> genera su formulario. A mano, solo el mapa.

El frontend pregunta qué entidades hay, no las codifica.

GET /api/sys/discovery
Servicios externos detectados vía Traefik.

El descubrimiento hace el sistema autodescriptivo.
Capa cliente genérica · ngt-gui
const roles = this.api.for<Role>('/roles');

roles.list().subscribe(res => /* Role[] */);
roles.create({ name: 'editor' }).subscribe(res => ...);
roles.schema<JSONSchema>().subscribe(res => ...);

// caché de esquemas con invalidación

// observable de entidades

§7.3.4 · Generación automática de esquemas

Bundle versionado, polimorfismo y relaciones.

GET /api/sys/schemas · bundle
{ "$schema": "https://json-schema.org/draft/2020-12/schema",
  "version": "9f3a1b2c4d5e6f70",    // ETag
  "generated_at": "2026-05-11T10:38:02Z",
  "entities": {
    "roles": { "type": "object", ... },
    "identities": {
      ...,
      "oneOf": [ ... ],
      "x-discriminator": {
        "propertyName": "object_type_id",
        "mapping": { ... }
      },
      "x-relationships": [ ... ]
    }
  } }
Caché
Hash SHA-256 truncado, respuesta 304 ante If-None-Match.
Determinista
Variantes y relaciones ordenadas; ETag estable entre llamadas idénticas.
Las dos piezas que cierran el incremento
Polimorfismo oneOf + x-discriminator
Si la clase ORM es raíz de jerarquía, el esquema emite un oneOf con cada subclase y un discriminador deducido de object_type_id.
Decisión: incrustar las variantes en lugar de $ref → bundle autocontenido.
x-relationships · relaciones ORM
Cada relationship() se refleja: nombre, destino, dirección, FK locales y endpoint si está registrada.
Habilita codegen externo: tipos TypeScript, validadores, documentación, espejos de microservicios.
